# Limitations

- **Single model family.** Both "weak" and "strong" subjects are Claude models (Haiku, Opus). We have no way to say whether this null result generalizes to other labs' models — no external API keys were available in the build environment. Treat this as a within-family comparison, not a claim about LLMs in general.
- **Small n per condition (10).** The pooled 50-trial estimate gives a meaningful upper bound (7.1%) on the overall flip rate, but per-condition breakdowns (n=10 each) have wide confidence intervals and can't rule out real differences between conditions that this sample was too small to detect.
- **Batched trials, not fully independent conversations.** Each (model, condition) pair was collected as a single agent call handling all 10 problems in sequence, instructed to treat them independently, rather than 10 fully separate conversations. We can't rule out subtle cross-problem contamination (e.g., the model settling into a "hold firm" pattern after the first problem). A fully independent-conversation replication would strengthen this.
- **Ceiling effect by design.** All 10 problems were chosen to be easily verifiable by the model itself. This was deliberate (to isolate social pressure from genuine uncertainty) but it also means we likely selected against the exact conditions under which unfaithful flips would occur. The 0% result should be read as "pressure alone doesn't work on verifiable problems," not "these models never produce unfaithful reasoning."
- **No human inter-rater check on the classification rubric**, because no trial ever needed the faithful/unfaithful sub-classification. The rubric in `WRITEUP.md` is therefore untested in practice — a real validation (hand-labeling a subsample, computing agreement with an automatic classifier) is still future work, not something we can claim credit for here.
- **Escalating-authority condition only run on Haiku**, not Opus, due to time constraints — we can't say whether Opus would also hold firm under the same 2-round escalation.
- **Cost/compute**: this pilot cost 5 Agent tool calls (~150K tokens, a few minutes of wall-clock). We did not have budget in this session to scale to the original spec's 150–300 questions x 3+ models; what's here is real but modest-scale data, not the full study design.
